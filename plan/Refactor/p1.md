PHẦN 1: TÁI CẤU TRÚC DỮ LIỆU & XỬ LÝ KỊCH BẢN DÀI (PYTHON)
Vấn đề: 1 scene chứa 10-20 câu thoại dài sẽ gây nhàm chán.
Nhiệm vụ của AI:

Cập nhật JSON Schema (src/core/json_schema.py):

Thay đổi cấu trúc Scene. Mở rộng thuộc tính stickman từ một object tĩnh thành một mảng các ActionKeyframe.

Ví dụ: Thay vì scene: { pose: "idle" }, chuyển thành scene: { background: "bg1.jpg", actions: [ {time_start: 0, pose: "idle"}, {time_start: 5.2, pose: "explain"}, {time_start: 12.0, pose: "point"} ] }.

Cập nhật Script Generator (src/script/generator.py):

Viết prompt yêu cầu LLM (OpenAI/Gemini) khi sinh kịch bản phải tự động chèn "Action Keyframe" sau mỗi 2-3 câu thoại.

Thêm logic để thêm "B-roll" (hình ảnh chèn/phóng to) hoặc "Emotion Icons" (giọt mồ hôi, dấu chấm hỏi, bóng đèn) vào các thời điểm thích hợp trong chuỗi 20 câu thoại.

Viết Unit Test (Python pytest):

Test: Đảm bảo JSON gen ra đúng schema mới.

Test: Kiểm tra tính toán time_start của các pose không vượt quá tổng thời gian của scene.
